# Nourished Rebel Color System

This document provides the complete color palette for the Nourished Rebel brand, including all color variants and usage guidelines.

## Overview

The Nourished Rebel color system is built around **#675C53** as the primary brand color. This warm brown serves as the anchor for the entire design system and should be the most prominent color throughout the UI.

All colors include variants from 50 (lightest) to 950 (darkest) for maximum flexibility while maintaining brand consistency.

---

## Primary Brand Color

### Warm Brown (#675C53)

**Primary use:** CTAs, H1/H2 headings, active states, primary buttons

This is the dominant brand color and must be used consistently across all primary interactive elements and major headings.

**Color Ramp:**
- 50: `#F8F7F6` - Subtle tints for hover states
- 100: `#F0EEEC` - Very light backgrounds
- 200: `#DED9D5` - Light backgrounds, borders
- 300: `#C3BAB3` - Muted states
- 400: `#A1938A` - Lighter text on dark backgrounds
- **500: `#675C53`** ← BASE COLOR
- 600: `#524741` - Hover states for buttons
- 700: `#423A34` - Pressed states
- 800: `#332D29` - Dark text options
- 900: `#282421` - Very dark variations
- 950: `#1A1816` - Maximum contrast

**Accessibility:**
- 500 on cream-200 (#EFE9E3): **AAA** (4.8:1 for normal text, 7.2:1 for large text)
- White text on 500: **AA** (4.5:1)
- White text on 600: **AAA** (7.1:1)

## Usage Guidelines

### DO:

✓ Use **brand-500 (#675C53)** as the primary color for all CTAs and major headings
✓ Alternate backgrounds between **cream-200 (#EFE9E3)** and **stone-300 (#D6CFC4)** for visual rhythm
✓ Use **charcoal-900 (#3F3A36)** for body text instead of pure black
✓ Maintain **AA or AAA** contrast ratios for accessibility
✓ Use supporting colors (olive, terracotta, clay) **sparingly** to let the primary color dominate
✓ Keep the overall palette warm, grounded, and cohesive
✓ Use lighter variants (50-300) for backgrounds and subtle accents
✓ Use darker variants (600-950) for text and high-contrast needs

### DO NOT:

✗ Use pure black (#000000) anywhere in the design
✗ Overuse the accent colors - they should accent, not dominate
✗ Use purple, indigo, or violet hues unless explicitly requested
✗ Mix this palette with colors outside the system
✗ Use insufficient contrast ratios that fail AA standards
✗ Use brand-500 on backgrounds that don't provide adequate contrast

## Accessibility Standards

All color combinations must meet at least **WCAG 2.1 Level AA** standards:

- **Normal text** (< 18pt): Minimum 4.5:1 contrast ratio
- **Large text** (≥ 18pt or ≥ 14pt bold): Minimum 3:1 contrast ratio
- **AAA standard** (preferred): 7:1 for normal text, 4.5:1 for large text

### Recommended Text/Background Combinations:

**High Contrast (AAA):**
- charcoal-900 on cream-200: 10.5:1
- charcoal-900 on stone-300: 9.2:1
- White on brand-600: 7.1:1

**Good Contrast (AA):**
- brand-500 on cream-200: 4.8:1
- charcoal-800 on cream-200: 8.5:1
- White on brand-500: 4.5:1

## Implementation

### Tailwind Config

Colors are defined in `tailwind.config.js`:

```javascript
colors: {
  brand: {
    primary: '#675C53',
    'primary-dark': '#524741',
  },
  background: {
    primary: '#EFE9E3',
    secondary: '#D6CFC4',
  },
  text: {
    primary: '#3F3A36',
    inverse: '#EFE9E3',
    heading: '#675C53',
  },
  accent: {
    muted: '#A6B640',
    secondary: '#C6866A',
    deep: '#C78553',
  },
}
```

### CSS Custom Properties

All color variants are available as CSS custom properties in `src/styles/tokens.css`:

```css
/* Example usage */
color: var(--color-brand-500);
background: var(--color-cream-200);
border-color: var(--color-stone-300);
```
